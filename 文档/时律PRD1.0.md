## 2. 项目背景与目标

### 2.1 项目背景
目标用户普遍存在以下问题：
- 有长期目标，但执行容易中断
- 中断后容易产生自责并放弃
- 需要低压力、可重启的执行机制，而非高压监督

《时律》MVP 聚焦“通过拆解与路径重构，让用户更容易达成目标”，通过“唯一长期目标 + 每日三任务 + 次日确认 + 语音调节”形成最小闭环。

### 2.2 项目目标
- 帮助用户在中断后仍能继续推进长期目标
- 通过拆解与路径重构，让用户更容易达成目标
- 降低每日执行决策成本
- 通过轻量反馈与策略调整，提高持续执行概率

### 2.3 用户画像与需求分析

#### 2.3.1 用户画像
- 核心人群：职场人、自由职业者、内容创作者
- 共性特征：
  1. 有长期目标，但执行连续性不足
  2. 能制定计划，但容易被临时事务打断
  3. 掉队后易产生挫败感，难以重启
- 使用偏好：希望被温和引导，不希望被高压监督

#### 2.3.2 关键需求分析
1. 低门槛启动：每天打开后能快速知道“先做什么”
2. 断点续接：中断后能无负担恢复执行，不因一次失败放弃
3. 任务适配：任务难度/类型可随状态动态调整
4. 结果可见：用户能看到进度变化与阶段推进，增强掌控感

### 2.4 MVP 成功标准（方向性）
- 用户能完成从建档到首日执行的闭环
- 用户在次日能完成昨日确认并进入新一天任务
- 用户可通过语音反馈调整任务难度/类型/内容

## 3. 商业化分析

### 3.1 商业化定位
《时律》定位为“执行恢复与目标达成辅助工具”，核心价值不在记录计划，而在于通过拆解与路径重构，让用户持续推进长期目标。

### 3.2 目标用户与付费动机
- 目标用户：有明确成长目标但执行易中断的职场人、自由职业者、内容创作者
- 核心付费动机：
  1. 需要更高成功率（而非更多计划功能）
  2. 需要低压力重启机制，减少“掉队后放弃”
  3. 希望任务难度/类型可随状态动态调整

### 3.3 收费模式（MVP 阶段建议）
1. 免费版（基础闭环）
   - 唯一长期目标
   - 每日三任务
   - 次日确认
2. 订阅版（增强能力）
   - 更高频语音策略调整
   - 更细颗粒度任务优化建议
   - 更完整历史分析与阶段复盘

### 3.4 核心商业指标（建议）
- 激活：建档完成率（完成 AI 五问并创建目标）
- 留存：D1/D7 留存、次日确认完成率
- 价值感：语音策略使用率、任务完成率提升幅度
- 变现：免费→订阅转化率、订阅续费率

### 3.5 商业化假设验证路径
1. 先验证“持续执行价值”是否成立（留存、完成率）
2. 再验证“策略调节价值”是否可感知（语音后完成率变化）
3. 最后验证“付费意愿”是否成立（增强能力转化）

### 3.6 Pro 转化路径（融资重点）
1. 用户在免费版完成基础闭环（建档→执行→次日确认）
2. 用户触发语音策略调节与自适应重排，感知“任务更贴合自己状态”
3. 系统展示“达成率提升对比”（如近 7 天完成率变化）
4. 在价值感知节点触发 Pro 引导（解锁更高频/更深度策略能力）
5. 通过续费率验证长期商业价值

## 4. 流程图（文本版）

### 4.1 全局分流流程
1. 用户打开 App → 进入启动页
2. 启动页完成初始化后判断：
   - 若无长期目标（`has_goal=false`）→ 新建入口页
   - 若有长期目标且需昨日确认（`need_yesterday_confirm=true`）→ 次日确认页
   - 若有长期目标且无需昨日确认 → 任务页

### 4.2 首次使用流程
1. 启动页
2. 新建入口页（文字/语音输入目标）
3. 长期目标澄清页（AI 五问选择题）
4. 生成周期与阶段名称
5. 用户确认
6. 生成今日三任务
7. 进入任务页

### 4.3 日常执行流程
1. 进入任务页
2. 用户对任务执行“完成/忽略”
3. 可“重新生成今日三任务”
4. 可通过语音输入调整任务策略（影响今日及后续）

### 4.4 次日前置流程
1. 次日首次打开且存在昨日未确认任务 → 进入次日确认页
2. 用户逐条选择“完成/忽略”（忽略需填写原因）
3. 全部确认后进入任务页

## 5. 功能清单（分级）

### 5.1 P0（MVP 必做）
1. 启动分流：按是否有目标/是否需次日确认进入对应页面
2. 新建入口：支持文字输入创建目标（语音输入可用）
3. AI 五问建档：选择题问答、生成周期与阶段名称（仅展示，不可编辑）
4. 任务页执行闭环：每日 3 任务、完成/忽略、重生成今日三任务
5. 语音策略调节：根据语音内容调整今日及后续任务难度/类型/内容，并展示调整说明
6. 次日确认：昨日任务逐条确认（完成/忽略）后放行今日任务
7. 日历页：展示进度、阶段区间、日期详情（阶段名称 + 当日任务）
8. 我的页基础能力：用户信息、数据摘要、设置与退出登录

### 5.2 P0.5（融资验证功能）
1. 未来 7 天自适应重排：基于完成/忽略/语音反馈，自动调整未来任务难度、类型与内容
2. 策略可解释卡片：每次重排后展示“为什么调整、调整了什么”
3. 达成率提升对比：展示开启自适应前后完成率变化（周维度）
4. 轻量订阅触发点：当用户看到提升对比后，提供 Pro 权益引导

### 5.3 P1（后续功能）
1. AI 偏好设置（含偏好完善提醒红点）
2. 成就体系与徽章展示
3. 深度完成统计与重构历史详情
4. 会员增强能力（深度 AI 偏好分析、全周期看板）

### 5.4 分级说明
- P0：MVP 上线必备，直接影响核心闭环（建档→执行→次日确认→持续执行）
- P0.5：用于融资阶段证明“策略有效 + 可转化付费”
- P1：在 P0/P0.5 跑通后迭代，不影响 MVP 核心可用性

## 6.3 新建入口页

### 6.3.1 页面设计
- 标题：开始你的长期目标
- 输入方式：文字（默认）+ 语音（可选）
- 主按钮：创建长期目标

### 6.3.2 功能需求
1. 支持用户输入目标意图
2. 支持语音输入并转文字回填，可编辑
3. 点击“创建长期目标”进入 AI 五问页
4. 已有长期目标时不展示本页，重定向任务页

### 6.3.3 规则与异常
- 语音识别失败：提示重试
- 麦克风未授权：提示授权
- 跳转失败：提示重试

## 6.4 长期目标澄清页（AI 五问）

### 6.4.1 页面设计
- 五问逐题展示，进度显示（1/5~5/5）
- 题型：选择题（单选/多选）
- 结果区：建议执行天数 + 结束日期 + 阶段名称

### 6.4.2 功能需求
1. AI 发起 5 个澄清问题（选择题）
2. 用户逐题作答并可回退修改
3. 基于选择结果生成执行周期与阶段名称
4. 阶段功能仅展示“阶段名称”，不支持用户修改阶段安排
5. 用户确认后创建长期目标并生成今日三任务
6. 创建成功后进入任务页

### 6.4.3 规则与异常
- 初始目标为空：不可开始
- 单选题：必须且仅可选 1 项
- 多选题：至少选 1 项（上限按配置）
- 修改前序题：后续题答案清空并重算建议
- 提交中按钮 loading，防重复提交
- 网络失败保留输入并提示重试

## 6.5 任务页（长期目标 + 每日三任务）

### 6.5.1 页面设计
- 顶部：长期目标摘要（标题、起止日期/剩余周期）
- 今日任务区：3 个任务卡片
- 任务动作：完成 / 忽略
- 语音策略区：语音输入按钮
- 调整说明区：展示“本次调整说明”
- 补充操作：重新生成今日三任务

### 6.5.2 功能需求

#### A. 长期目标管理
1. 查看当前唯一长期目标
2. 删除长期目标（删除后返回新建入口页）

#### B. 每日三任务管理
1. 拉取并展示当日 3 个小任务
2. 任务支持“完成”
3. 任务支持“忽略”
4. 忽略时收集原因（太难了/不想做/不相关/时间不够/其他）
5. 支持重新生成今日三任务
6. 重生成前 AI 询问原因并记录

#### C. 语音调节任务策略
1. 用户可语音输入近况/偏好
2. 系统转写并允许用户确认/编辑
3. AI 基于语义调整任务策略
4. 调整范围覆盖“今日任务 + 后续每日任务”
5. 调整维度包括：难度 / 类型 / 内容侧重
6. 每次调整后展示“本次调整说明”

### 6.5.3 规则与异常
- 若无长期目标：自动跳新建入口页
- 任务完成采用乐观更新，失败则回滚
- 忽略“其他”需补充文本
- 语音识别失败可重录或改文字
- AI 调整失败保留原任务并提示重试

## 6.6 日历页

### 6.6.1 页面设计（页面交互结构）
页面仅保留 3 个板块：

1. 进度条板块
- 展示长期目标总进度
- 文案示例：已完成 8 / 21 天
- 进度条短展示即可，不展开更多信息

2. 日历板块
- 按月展示执行周期内的日期
- 不同阶段用不同颜色覆盖日期区间
- 阶段颜色由系统配置，不支持用户自定义或修改
- 示例：
  - 阶段1：黄色
  - 阶段2：绿色
  - 阶段3：蓝色
- 已完成的日子统一加深，标黑显示
- 当前选中的日期有高亮态

3. 下方详情板块
- 点击日历中的某一天后，展示：
  - 阶段名称
  - 该日的 3 个任务
- 列表仅做展示，不承载复杂操作

### 6.6.2 功能需求
1. 展示长期目标的总进度
2. 展示执行周期内的日历
3. 按阶段用不同颜色覆盖对应日期
4. 已完成日期显示为深色/标黑
5. 支持点击某一天查看详情
6. 下方展示“阶段名称 + 该日三个任务”
7. 默认进入页面时选中今天

### 6.6.3 交互逻辑
1. 进入页面后，默认展示：
   - 顶部进度条
   - 中部日历
   - 下方今日详情
2. 系统默认选中今天，并拉取今天所在阶段与当天 3 个任务
3. 用户点击日历中的某一天后：
   - 该日期进入选中态
   - 下方详情区更新为该日内容
4. 系统根据该日期所属区间，自动显示对应阶段名称
5. 系统根据该日期任务完成情况，决定日期是否标黑
6. 若该日已有任务，则下方展示该日 3 个任务
7. 若该日无任务，则下方展示空状态：暂无任务

异常处理：
- 页面加载失败：提示“加载失败，请重试”
- 某日任务加载失败：下方提示“任务加载失败，请重试”

## 6.7 我的页（个人中心）

### 6.7.1 页面设计
- 页面类型：滚动页（ScrollView），无固定顶部栏
- 顶部操作区：左上设置图标、右上福利图标
- 用户信息区：头像、昵称、UID（带复制）
- 数据统计区：进行中、完成率、连续打卡
- 会员横幅区：未开通用户展示，已开通用户隐藏或替换为有效期信息
- 功能宫格区（3 项）：我的成就、完成统计、重构历史
- 功能列表区（分组）：账号设置、工具、通用、退出登录

### 6.7.2 功能需求

#### A. 顶部操作栏
1. 点击设置图标进入设置页
2. 点击福利图标进入福利中心（或活动弹窗）
3. 两个图标点击热区不小于 44×44px

#### B. 用户信息区
1. 头像展示并支持更换
2. 昵称展示，支持进入编辑资料
3. UID 展示并支持一键复制
4. 头像上传流程支持拍照/相册选择 + 裁剪 + 上传

#### C. 数据统计栏
1. 展示进行中任务数量
2. 展示整体完成率（已完成天数 ÷ 总计划天数）
3. 展示连续打卡天数
4. 数据为展示态，不支持点击跳转

#### D. 会员横幅
1. 未开通用户展示「立即开通」入口
2. 已开通用户隐藏横幅或展示会员有效期
3. 点击横幅任意区域可进入会员购买页

#### E. 功能宫格
1. 我的成就 → 成就页
2. 完成统计 → 统计详情页
3. 重构历史 → 历次 AI 重构记录页

#### F. 功能列表
1. 账号设置：编辑资料、通知设置
2. 工具：草稿箱、同步数据
3. 通用：外观设置、语言设置、清除缓存
4. 退出登录：底部红色按钮，点击后二次确认

### 6.7.3 交互逻辑
1. 页面进入即拉取用户信息、统计数据、会员状态
2. 点击头像：ActionSheet（拍照/相册/取消）→ 裁剪 → 上传 → 更新头像
3. 点击 UID 复制：复制成功后 Toast「已复制」
4. 列表行点击反馈：背景短暂高亮（约 100ms）后跳转
5. 点击清除缓存：弹窗确认后执行清除并 Toast「缓存已清除」
6. 点击退出登录：弹窗确认后清理本地 token，跳转登录页（replace）

### 6.7.4 规则与异常
- 未登录进入我的页：直接跳转登录页
- 用户数据加载失败：统计位展示 `--`，提示「数据加载失败」
- 头像上传失败/超时：保留原头像并提示重试
- 完成率或连续打卡为 0 时正常展示 `0%` / `0`
- 退出登录后本地任务数据保留，再次登录同账号可恢复

### 6.7.5 与底部导航对齐说明
- 当前 MVP 底部导航固定为：任务 / 日历 / 我的
- AI 偏好功能不在 MVP 范围内，后续版本可通过我的页入口补充

## 6.9 全局规则
1. 同一账号同一时间仅允许 1 个长期目标
2. 长期目标删除后，当日三任务同步失效/清空
3. 支持在不改长期目标前提下重生成当日三任务
4. 次日首次打开且有昨日未确认任务时，必须先确认再进入任务页
5. 每日任务默认固定为 3 条

## 7. 数据采集需求

## 7.1 采集目标
建立从“建档→执行→次日确认→持续执行”的基础数据闭环，为后续优化提供依据。

## 7.2 事件采集清单（通用方案）

### 7.2.1 启动与分流
- `app_launch`
- `launch_route_decided`
  - 参数：`has_goal`、`need_yesterday_confirm`、`target_page`

### 7.2.2 建档流程
- `goal_create_entry_click`
- `goal_input_voice_used`
- `goal_input_voice_transcribe_success`
- `ai_question_answered`
  - 参数：`question_index`、`question_type(single|multi)`、`selected_option_count`
- `goal_plan_generated`
  - 参数：`suggested_days`、`phase_count`
- `goal_created_success`

### 7.2.3 任务执行
- `task_list_loaded`
- `task_completed`
  - 参数：`task_id`、`task_type`
- `task_ignored`
  - 参数：`task_id`、`reason`、`reason_text_length`
- `tasks_regenerated`
  - 参数：`reason`

### 7.2.4 语音策略调节
- `task_strategy_voice_start`
- `task_strategy_voice_transcribe_success`
- `task_strategy_adjust_submitted`
- `task_strategy_adjust_applied`
  - 参数：`difficulty_change`、`type_change`、`content_focus_change`

### 7.2.5 次日确认
- `yesterday_confirm_page_show`
- `yesterday_task_confirmed`
  - 参数：`result(done|ignored)`、`ignore_reason`
- `yesterday_confirm_completed`

## 7.3 关键统计口径（建议）
- 建档完成率 = `goal_created_success / app_launch(无目标用户)`
- 首日任务完成率 = 首日 `task_completed` 用户数 / 首日建档成功用户数
- 次日确认完成率 = `yesterday_confirm_completed / yesterday_confirm_page_show`
- 任务忽略率 = `task_ignored / (task_completed + task_ignored)`
- 语音策略使用率 = `task_strategy_adjust_submitted / 任务页活跃用户数`

## 7.4 投资人指标看板（建议）
1. Pro 转化率：`pro_paid_users / pro_offer_exposed_users`
2. Pro 续费率：`renewed_users / expiring_pro_users`
3. 自适应有效率：使用自适应后 7 日完成率提升的用户占比
4. 价值感知率：查看“达成率提升对比”后继续执行的用户占比
5. 数据壁垒强度：人均有效反馈次数（语音调节 + 忽略原因 + 次日确认）

## 8. 非功能性需求

## 8.1 性能
1. 启动页分流判定应在可接受时间内完成，避免长时间白屏
2. 任务页核心内容（目标 + 今日三任务）应优先加载
3. 语音转写与策略调整应提供处理中反馈，避免用户无感等待

## 8.2 可用性与容错
1. 所有关键提交操作需防重复点击
2. 网络失败时需保留用户输入并支持重试
3. 异常提示文案需清晰、可执行（告诉用户下一步）

## 8.3 防呆设计
1. 关键破坏性操作必须二次确认（如删除长期目标）
2. 表单与题目未完成时，主按钮置灰并给出明确原因提示
3. 语音输入失败时提供一键重试与改用文字输入入口
4. 选择“其他”原因时必须补充文本，未填写不可提交
5. 次日确认存在未处理任务时，禁止放行进入任务页
6. 提交成功前不得显示最终成功态，避免误导用户

## 8.4 数据一致性
1. 任务状态变更需保证前后端一致
2. 乐观更新失败必须回滚，避免“显示成功但实际失败”
3. 次日确认未完成前不得放行至任务页

## 8.5 安全与隐私（语音相关）
1. 麦克风权限需显式授权
2. 语音内容仅用于转写与任务策略调整
3. 用户可选择不用语音功能，不影响核心流程

## 8.6 可扩展性
1. AI 五问题目与选项需支持配置化
2. 任务生成策略需支持后续迭代（不改动主流程）
3. 埋点事件命名需稳定，支持版本演进

## 9. 接口文档（MVP）

### 9.1 目标与约定
- 接口风格：REST
- 数据格式：`application/json`
- 鉴权方式：登录态 Token（具体方案由服务端统一）
- 时间格式：ISO 8601（如 `2026-04-04T10:00:00+08:00`）

### 9.2 通用响应结构（建议）
- 成功：
  - `code`: 0
  - `message`: "ok"
  - `data`: 业务数据
- 失败：
  - `code`: 非 0
  - `message`: 错误描述
  - `data`: 可为空

### 9.3 核心接口清单

#### 9.3.1 启动分流
1. `GET /api/v1/app/bootstrap`
   - 用途：获取启动分流所需状态
   - 返回关键字段：
     - `has_goal`（是否已有长期目标）
     - `need_yesterday_confirm`（是否需次日确认）

#### 9.3.2 目标建档
2. `POST /api/v1/goals/draft`
   - 用途：提交新建入口页初始目标（文字/语音转写）
   - 请求字段：`goal_text`

3. `GET /api/v1/goals/questions`
   - 用途：获取 AI 五问题目配置（单选/多选）

4. `POST /api/v1/goals/plan/preview`
   - 用途：提交五问答案，生成周期与阶段名称预览
   - 请求字段：`answers[]`

5. `POST /api/v1/goals`
   - 用途：确认创建长期目标并生成今日三任务
   - 请求字段：`goal_text`、`answers[]`、`plan_preview_id`

#### 9.3.3 任务页
6. `GET /api/v1/tasks/today`
   - 用途：获取今日 3 个任务与目标摘要

7. `POST /api/v1/tasks/{task_id}/complete`
   - 用途：提交任务完成

8. `POST /api/v1/tasks/{task_id}/ignore`
   - 用途：提交任务忽略与原因
   - 请求字段：`reason`、`reason_text`（可选）

9. `POST /api/v1/tasks/regenerate`
   - 用途：重新生成今日三任务
   - 请求字段：`reason`、`reason_text`（可选）

10. `POST /api/v1/tasks/strategy/voice`
   - 用途：提交语音策略输入并触发任务策略调整
   - 请求字段：`voice_text`（语音转写结果）
   - 返回关键字段：`adjust_summary`（本次调整说明）

11. `DELETE /api/v1/goals/current`
   - 用途：删除当前长期目标（并清空当日任务）

#### 9.3.4 次日确认
12. `GET /api/v1/tasks/yesterday/pending`
   - 用途：获取昨日未确认任务列表

13. `POST /api/v1/tasks/yesterday/confirm`
   - 用途：提交昨日任务逐条确认结果
   - 请求字段：`items[]`（每项含 `task_id`、`result(done|ignored)`、`reason`、`reason_text`）

#### 9.3.5 日历与我的
14. `GET /api/v1/calendar`
   - 用途：获取执行周期日历数据（阶段颜色区间、日期完成状态）

15. `GET /api/v1/calendar/{date}`
   - 用途：获取指定日期详情（阶段名称 + 当日三任务）

16. `GET /api/v1/me/summary`
   - 用途：获取“我的”页摘要信息（目标摘要、累计完成数、连续执行天数）

### 9.4 错误码建议（示例）
- `1001`：参数缺失或不合法
- `1002`：鉴权失败或登录态失效
- `2001`：目标不存在
- `2002`：任务不存在
- `3001`：语音转写内容为空
- `5000`：服务内部错误

### 9.5 接口与规则对齐说明
1. 同一账号仅允许存在 1 个长期目标
2. 次日确认未完成前，不放行任务页核心数据
3. 任务默认每日固定 3 条，重生成后覆盖当日任务
4. 阶段信息仅返回“阶段名称”，不提供用户改阶段接口
