# 《时律》接口文档（MVP）

> 版本：1.0
>
> 适用范围：MVP 阶段全部前后端交互接口

---

## 1. 通用约定

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 鉴权方式 | Bearer Token（Header: `Authorization: Bearer <token>`） |
| 游客模式 | 使用设备级临时 Token，首次启动自动获取 |

### 1.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 0=成功，非0=失败（见错误码表） |
| message | string | 结果描述 |
| data | object/null | 业务数据，失败时为 null |

### 1.3 错误码

| code | 说明 |
|------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | Token 无效或过期 |
| 1003 | 无权限 |
| 2001 | 目标不存在 |
| 2002 | 已存在活跃目标（不可重复创建） |
| 2003 | 任务不存在 |
| 3001 | AI 服务调用失败 |
| 3002 | AI 服务超时 |
| 9999 | 服务器内部错误 |

## 2. 账号与鉴权

### 2.1 游客初始化

获取设备级临时 Token，无需用户手动登录。

```
POST /api/auth/guest-init
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| device_id | string | 是 | 设备唯一标识 |
| platform | string | 是 | ios / android |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | 访问令牌 |
| user_id | string | 用户 ID |
| is_guest | boolean | 是否游客 |

---

### 2.2 登录

游客转正式账号。

```
POST /api/auth/login
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| method | string | 是 | 登录方式（phone / wechat） |
| phone | string | 条件 | 手机号（method=phone 时必填） |
| code | string | 条件 | 验证码（method=phone 时必填） |
| wechat_code | string | 条件 | 微信授权码（method=wechat 时必填） |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | 新的访问令牌 |
| user_id | string | 用户 ID |
| nickname | string | 昵称 |
| avatar_url | string | 头像 URL |
| is_guest | boolean | false |

---

### 2.3 退出登录

```
POST /api/auth/logout
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | 降级为游客的临时 Token |

---

## 3. 启动分流

### 3.1 获取启动状态

App 启动后调用，决定路由跳转方向。

```
GET /api/launch/status
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| has_goal | boolean | 是否有活跃长期目标 |
| need_yesterday_confirm | boolean | 是否需要次日确认 |

**前端路由规则**

| has_goal | need_yesterday_confirm | 目标页面 |
|----------|----------------------|---------|
| false | - | 新建入口页 |
| true | true | 次日确认页 |
| true | false | 任务页 |

---

## 4. 长期目标

### 4.1 创建长期目标（启动 AI 建档）

用户输入目标意图后，创建目标并触发 AI 五问。

```
POST /api/goal/create
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 目标意图文本 |
| source | string | 否 | 输入来源（text） |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| goal_id | string | 目标 ID |
| first_question | object | 第一道 AI 澄清问题（见 4.2 question 结构） |

---

### 4.2 获取下一道澄清问题

提交上一题答案后，AI 根据上下文生成下一题。

```
POST /api/goal/{goal_id}/clarify/next
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question_index | number | 是 | 当前题序号（1-5） |
| selected_option | string | 是 | 选中的选项 ID（A/B/C/D） |
| custom_text | string | 条件 | 选 D 时的自定义文本 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| question_index | number | 下一题序号 |
| title | string | 问题标题 |
| description | string | 问题描述 |
| options | array | 选项列表 |
| options[].id | string | 选项 ID（A/B/C/D） |
| options[].label | string | 选项标签 |
| options[].description | string | 选项说明 |

---

### 4.3 提交澄清问答并生成规划

提交第 5 题答案后，AI 生成执行方案。

```
POST /api/goal/{goal_id}/clarify/submit
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question_index | number | 是 | 5（最后一题） |
| selected_option | string | 是 | 选中的选项 ID |
| custom_text | string | 条件 | 选 D 时的自定义文本 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| total_days | number | 总执行天数 |
| start_date | string | 开始日期（YYYY-MM-DD） |
| end_date | string | 结束日期（YYYY-MM-DD） |
| phases | array | 阶段列表 |
| phases[].name | string | 阶段名称 |
| phases[].days | string | 天数描述（如"第1-8天"） |
| phases[].description | string | 阶段说明 |
| phases[].color | string | 阶段颜色（HEX） |
| today_tasks | array | 今日三任务标题列表 |

---

### 4.4 确认规划并开始执行

用户在方案结果页点击"开始执行"。

```
POST /api/goal/{goal_id}/confirm
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 确认成功 |

---

### 4.5 查看当前长期目标

```
GET /api/goal/current
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| goal_id | string | 目标 ID |
| title | string | 目标标题 |
| total_days | number | 总天数 |
| start_date | string | 开始日期 |
| end_date | string | 结束日期 |
| current_day | number | 当前是第几天 |
| remaining_days | number | 剩余天数 |
| completion_rate | number | 完成率（0-100） |
| phases | array | 阶段列表（同 4.3） |

---

### 4.6 删除长期目标

```
DELETE /api/goal/{goal_id}
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 删除成功 |

> 删除后当日任务同步清空，前端跳转新建入口页。

## 5. 每日任务

### 5.1 获取今日任务

```
GET /api/tasks/today
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks | array | 今日三任务 |
| tasks[].id | string | 任务 ID |
| tasks[].title | string | 任务标题 |
| tasks[].icon | string | 图标名称 |
| tasks[].priority | string | 优先级（HIGH / MID / FOCUS） |
| tasks[].done | boolean | 是否已完成 |
| adjustment_note | string/null | AI 调整说明（有调整时返回） |

---

### 5.2 完成任务

```
POST /api/tasks/{task_id}/complete
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作成功 |

> 前端使用乐观更新，失败时回滚。

---

### 5.3 忽略任务

```
POST /api/tasks/{task_id}/ignore
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reason | string | 是 | 忽略原因（太难了/不想做/不相关/时间不够/其他） |
| reason_text | string | 条件 | 选"其他"时的补充说明 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作成功 |

---

### 5.4 重新生成今日任务

```
POST /api/tasks/regenerate
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reason | string | 是 | 重生成原因（任务太难了/不够有挑战/想换个方向/时间冲突/其他） |
| reason_text | string | 条件 | 选"其他"时的补充说明 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks | array | 新的三任务（结构同 5.1） |
| adjustment_note | string | AI 调整说明 |

---

## 6. 次日确认

### 6.1 获取待确认任务

```
GET /api/yesterday/tasks
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks | array | 待确认任务列表 |
| tasks[].id | string | 任务 ID |
| tasks[].title | string | 任务标题 |
| tasks[].icon | string | 图标名称 |
| tasks[].category | string | 任务分类 |
| tasks[].subtitle | string | 补充说明 |
| tasks[].resolved | boolean | 是否已确认 |
| tasks[].action | string/null | 确认结果（done/ignore，未确认时为 null） |

---

### 6.2 确认单个任务

```
POST /api/yesterday/tasks/{task_id}/resolve
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 是 | 确认结果（done / ignore） |
| ignore_reason | string | 条件 | action=ignore 时的原因 |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 操作成功 |

---

### 6.3 提交全部确认

全部任务确认后，提交并进入今日任务。

```
POST /api/yesterday/confirm
```

**请求参数**：无

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 确认成功 |

> 全部任务未确认完时返回错误（code: 1001）。

---

```
GET /api/calendar/month
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |

| tasks | array | 当日任务列表 |

| tasks[].icon | string | 图标名称 |

## 9. 埋点事件

前端通过统一接口上报埋点事件。

### 9.1 上报事件

```
POST /api/analytics/event
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| event_name | string | 是 | 事件名称（见 PRD 7.2 事件清单） |
| params | object | 否 | 事件参数 |
| timestamp | number | 是 | 客户端时间戳（ms） |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 上报成功 |

### 9.2 批量上报

```
POST /api/analytics/batch
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| events | array | 是 | 事件列表（每项结构同 9.1） |

**响应 data**

| 字段 | 类型 | 说明 |
|------|------|------|
| received | number | 成功接收条数 |
