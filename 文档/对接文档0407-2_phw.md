## 2. 联调基础信息

### 2.1 后端启动

后端项目目录：

- `backend/`

本地启动命令：

```bash
cd backend
npm run mysql:local:start
npm run dev
```

默认本地地址：

```text
http://127.0.0.1:4000
```

如果需要生产或测试环境地址，前端只需要替换 `baseURL`。

### 2.2 鉴权方式

除以下开放接口外，其余接口都需要带：

```http
Authorization: Bearer <token>
```

开放接口：

- `POST /api/auth/guest-init`
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /health`

### 2.3 统一响应格式

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败：

```json
{
  "code": 1001,
  "message": "参数错误",
  "data": null
}
```

前端建议统一判断：

1. 先看 HTTP 状态码
2. 再看 `code`
3. 成功时取 `data`
4. 失败时按错误码做业务分支

## 3. 前端主流程调用顺序

### 3.1 首次启动 / 游客态

1. 调 `POST /api/auth/guest-init`
2. 保存返回的 `token`
3. 调 `GET /api/launch/status`
4. 根据返回结果跳转：

- `has_goal=false`：进入新建入口页
- `has_goal=true && need_yesterday_confirm=true`：进入次日确认页
- `has_goal=true && need_yesterday_confirm=false`：进入任务页

### 3.2 创建目标

1. 调 `POST /api/goal/create`
2. 使用返回的 `first_question`
3. 前 4 题每题提交到 `POST /api/goal/{goal_id}/clarify/next`
4. 第 5 题提交到 `POST /api/goal/{goal_id}/clarify/submit`
5. 展示规划结果页
6. 点击开始执行后调 `POST /api/goal/{goal_id}/confirm`
7. 进入任务页

### 3.3 日常任务

进入任务页后：

1. 调 `GET /api/tasks/today`
2. 完成任务调 `POST /api/tasks/{task_id}/complete`
3. 忽略任务调 `POST /api/tasks/{task_id}/ignore`
4. 重生成任务调 `POST /api/tasks/regenerate`
5. 语音调节时先调 `POST /api/voice/transcribe`
6. 用户确认文本后调 `POST /api/tasks/strategy-adjust`

### 3.4 次日确认

当 `GET /api/launch/status` 返回 `need_yesterday_confirm=true` 时：

1. 调 `GET /api/yesterday/tasks`
2. 对每条任务调 `POST /api/yesterday/tasks/{task_id}/resolve`
3. 全部处理完后调 `POST /api/yesterday/confirm`
4. 再进入任务页并调 `GET /api/tasks/today`

注意：

- 当前后端已按 PRD 加门禁
- 当存在昨日未确认任务时，`GET /api/tasks/today` 不再放行，会返回错误

## 4. 接口对接重点

### 4.1 账号与 Token

#### `POST /api/auth/guest-init`

用途：

- 初始化游客身份

关键返回：

- `token`
- `user_id`
- `is_guest`

前端动作：

- 首次拿到后立刻保存 `token`

#### `POST /api/auth/register`

用途：

- 创建正式账号
- 如当前处于游客态，可把当前游客身份升级成正式账号

请求参数：

- `account`
- `password`
- `nickname` 可选

关键注意：

- 注册成功后会返回新 `token`
- 前端必须用新 token 覆盖旧游客 token
- `account` 仅支持字母/数字/下划线，长度 4-24
- `password` 长度 6-64

#### `POST /api/auth/login`

用途：

- 使用账号密码登录正式账号

请求参数：

- `account`
- `password`

关键注意：

- 登录成功后会返回一个新 `token`
- 前端必须用新 token 覆盖旧 token
- 当前不再使用微信登录
- 当前不再使用手机号验证码登录

#### `POST /api/auth/logout`

用途：

- 退出登录并降级为游客

关键注意：

- 返回的是新的游客 `token`
- 前端必须覆盖本地旧 token
- 旧登录 token 会失效

### 4.2 长期目标

#### `POST /api/goal/create`

关键返回：

- `goal_id`
- `first_question`

#### `POST /api/goal/{goal_id}/clarify/next`

关键注意：

- `question_index` 传当前题号
- `selected_option` 只能是 `A/B/C/D`
- 当选 `D` 时必须传 `custom_text`

#### `POST /api/goal/{goal_id}/clarify/submit`

关键返回：

- `total_days`
- `start_date`
- `end_date`
- `phases`
- `today_tasks`

#### `POST /api/goal/{goal_id}/confirm`

关键返回：

- `success: true`

#### `GET /api/goal/current`

可用于任务页顶部和长期目标摘要展示。

#### `DELETE /api/goal/{goal_id}`

关键注意：

- 删除后当前目标失效
- 再调 `GET /api/launch/status` 会返回 `has_goal=false`
- 前端应跳回新建入口页

### 4.3 今日任务

#### `GET /api/tasks/today`

关键返回：

- `tasks` 固定 3 条
- `adjustment_note`

任务项结构：

- `id`
- `title`
- `icon`
- `priority`
- `done`

关键注意：

- 若存在昨日未确认任务，该接口会返回 `1001`
- 前端应跳次日确认页，而不是继续展示今日任务

#### `POST /api/tasks/{task_id}/complete`

关键注意：

- 前端可做乐观更新
- 失败要回滚

#### `POST /api/tasks/{task_id}/ignore`

关键注意：

- `reason` 必填
- 当 `reason=其他` 时，`reason_text` 必填

#### `POST /api/tasks/regenerate`

关键返回：

- 新的 `tasks`
- `adjustment_note`

### 4.4 语音能力

#### `POST /api/voice/transcribe`

上传方式：

- `multipart/form-data`

字段：

- `audio`: 文件
- `format`: 可选，建议传 `m4a/mp3/wav`

关键返回：

- `text`
- `confidence`

#### `POST /api/tasks/strategy-adjust`

关键返回：

- `tasks`
- `adjustment_note`
- `adjustments`

关键注意：

- 该接口会影响今日任务以及后续任务
- 前端至少需要立即刷新今日任务展示

### 4.5 次日确认

#### `GET /api/yesterday/tasks`

关键返回：

- `tasks[].resolved`
- `tasks[].action`

#### `POST /api/yesterday/tasks/{task_id}/resolve`

关键注意：

- `action` 只能是 `done / ignore`
- `action=ignore` 时 `ignore_reason` 必填

#### `POST /api/yesterday/confirm`

关键注意：

- 若还有未处理项，会返回 `1001`
- 全部确认后才会成功

### 4.6 日历和我的页

#### `GET /api/calendar/month`

用于：

- 月历渲染
- 进度条数据
- 阶段颜色展示

#### `GET /api/calendar/day/{date}`

用于：

- 点击某一天后的详情区

#### `GET /api/user/profile`

当前字段语义已和 PRD 对齐：

- `stats.ongoing`：今日未完成任务数
- `stats.completion_rate`：完成率
- `stats.streak`：连续打卡天数

## 5. 前端必须注意的业务规则

### 5.1 同时只能有一个长期目标

若已有活跃目标，再调用创建接口会返回：

- HTTP `409`
- `code=2002`

前端建议：

- 进入新建页前，先以 `launch/status` 为准
- 如仍收到 `2002`，提示用户当前已有目标并回到任务页

### 5.2 有昨日未确认任务时，不能进入今日任务

这是当前最重要的门禁规则之一。

表现：

- `launch/status.need_yesterday_confirm = true`
- `tasks/today` 会被拒绝

前端建议：

- 启动页和任务页进入前都以 `launch/status` 为准
- 若任务页请求返回 `1001` 且提示“请先完成昨日确认”，直接跳次日确认页

### 5.3 Token 更新必须覆盖

以下两个接口都会返回新 token：

- `register`
- `login`
- `logout`

前端如果不覆盖，会立刻出现 `1002 Token 无效或过期`

### 5.4 “其他”类原因必须补文本

涉及接口：

- `POST /api/tasks/{task_id}/ignore`
- `POST /api/tasks/regenerate`
- `POST /api/goal/{goal_id}/clarify/next`

条件：

- 选“其他”或 `D` 时必须补文本

## 6. 错误码对接建议

建议前端按下面方式处理：

- `1001`：参数错误 / 当前状态不允许操作，展示后端 message
- `1002`：token 失效，清 token 并回游客初始化或登录流
- `2001`：目标不存在，通常跳回新建入口页
- `2002`：已存在目标，跳任务页
- `2003`：任务不存在，刷新任务列表
- `2004`：账号已存在，提示更换账号
- `2005`：账号或密码错误，提示用户检查输入
- `3001/3002/3003`：AI/ASR 失败，提示重试并保留当前页面输入
- `9999`：统一兜底错误提示

## 7. 建议前端联调检查清单

前端切真实接口时，建议至少验证下面这些：

1. 游客初始化后 token 是否正确保存
2. 注册后是否覆盖旧游客 token
3. 登录后是否覆盖旧 token
4. 登出后是否覆盖为新游客 token
5. 创建目标后五问是否能完整走通
6. 第 5 题提交后是否能正确展示规划结果
7. 任务页是否稳定只展示 3 条任务
8. 忽略任务时“其他”是否要求补充文本
9. 次日确认未完成时是否无法进入今日任务
10. 日历点击某一天后是否能刷新详情
11. 我的页统计字段是否直接取接口，不再本地推导
